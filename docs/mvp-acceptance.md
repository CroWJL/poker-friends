# MVP 验收清单（德州扑克）

本文档用于验证 Poker Friends 当前 MVP 是否达到“可演示、可联调”状态。

## 1. 验收目标

最小闭环必须可用：

- 创建房间
- 加入房间
- WebSocket 建连成功
- 关键动作校验与广播可用
- 回合可从 `PREFLOP` 推进到 `FLOP`

## 2. 前置条件

- 后端服务运行在 `http://localhost:8080`
- 已安装 Node.js（建议 18+）
- 项目依赖已安装：`pnpm install`

## 3. 自动化验收（推荐）

项目根目录执行：

```bash
pnpm acceptance:mvp
```

等价命令：

```bash
node mvp-acceptance.mjs
```

脚本会自动完成：

- 创建房间（Alice）
- 加入房间（Bob）
- 双客户端 WS 建连并接收快照
- 验证非法 `CHECK` 被拒绝
- 验证 `CALL` 后快照广播
- 验证回合推进到 `FLOP`
- 验证快照包含 `actionPlayerId`

通过标准：

- 输出中 `summary.fail` 必须为 `0`

## 4. 手动验收（可视化）

1) 启动后端  
2) 启动前端：`pnpm dev:web`  
3) 浏览器打开 `http://localhost:5173`（或 Vite 提示端口）  
4) 页面 A 创建房间，页面 B 输入房间号加入  
5) 观察两端：

- 都能收到牌桌快照
- 当前行动玩家显示正确
- 非当前玩家动作按钮应禁用
- 当前玩家执行动作后，双方快照同步更新

## 5. 当前已覆盖规则

- 盲注初始化（第二位玩家入桌后自动下盲）
- 非当前玩家行动拒绝
- 非法 `CHECK` 拒绝
- 非法 `RAISE` 拒绝
- `CALL -> CHECK` 触发阶段推进到 `FLOP`
- Heads-up 下 `FOLD` 直接结束牌局
- `ALL_IN` 超过当前注会抬高 `currentBet`

## 6. 已知限制（MVP 范围内可接受）

- 仍为“临时玩家”模型，无完整用户体系
- 牌桌状态仍以内存为主，未做持久化恢复
- 无重连恢复与断线续局机制
